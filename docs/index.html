Tarot

📜 프로젝트 개요

이 프로젝트는 타로 카드 리딩을 통해 사용자가 자신의 질문에 대해 통찰을 얻을 수 있도록 지원하는 AI 기반 웹 애플리케이션입니다. 단순한 카드 선택을 넘어, 의미 있는 해석을 제공하는 것을 목표로 개발했습니다.

🃏 사용 카드 덱

  • 본 프로젝트는 Rider–Waite–Smith Tarot 덱(public)을 참고하여 카드 구조 및 의미를 기반으로 리딩을 제공합니다.
  • 1909년에 최초 출간된 Rider–Waite–Smith Tarot 덱은 현대 타로 리딩의 표준 이미지를 확립한 가장 널리 사용되는 대표적인 덱입니다.

🙋 내 역할

  • 전체 기획 및 개발 (1인 개발)
  • 프론트엔드와 백엔드 설계 및 구현
  • 배포 및 운영 환경 구축

🛠️ 기술 스택 및 환경

  • Frontend: HTML5, CSS3, JavaScript (Vanilla)
  • Backend: Node.js (Express)
  • Database: MariaDB
  • Web Server: Nginx (Reverse Proxy)
  • Hosting: Cafe24 (main), GitHub Pages (docs)

🚀 주요 기능

  • 카드 3장 무작위 셔플 및 선택 기능
  • AI 기반 리딩 결과 생성 및 출력
  • 로딩 애니메이션 및 다크모드 지원
  • 모바일 대응 반응형 레이아웃

🔗 시스템 구조

사용자가 질문을 입력하고 카드를 선택하면, 선택된 카드와 질문이 서버로 전송됩니다. 서버는 카드 정보와 질문을 바탕으로 AI 기반 리딩을 생성하고, 클라이언트에 결과를 반환하여 화면에 표시합니다.

🔥 어려웠던 점 & 해결 방법

SSH 인증서를 처음 설정하는 중 PEM 파일을 생성(cat)하여 하나의 인증서로 만들어서 연결하는 도중 PEM 파일의 `-----BEGIN PRIVATE KEY-----`와 `-----END PRIVATE KEY-----` 구문이 한 줄로 붙어버리는 문제로 인증하는데 어려움을 겪었습니다. 파일을 열어보기 전 까지 왜 안되지 순서가 잘못됐나 했었지만, 결국 이러한 현상을 발견하면서 그 두 구문을 개행처리 시킴으로 정상적으로 SSH 설정을 마무리할 수 있었습니다.
